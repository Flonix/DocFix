What is Crossreferencing ?

Cross references

From time to time it may make sense to add a link / reference to some other sections or chapters of your documentation. DocFix is able to handle refrences to targets inside any of the defined content files. Use the following examples and descriptions to set up a working reference.

Setting up a cross reference destination

First of all, a destination can be every html tag used in your documentation. The only thing that matters is the value of the id attribute.

Please read "How crossreferencing works" to make sure your destination is really found in the document. 

destination content 
...

All defined destinations can be directly reached from outside your DocFix documentation. Look here for complete explanation.

There is one destination called "all" which is not usable. (It will be overwritten by DocFix itself.) If this destination is requested, DocFix will generate one complete document and showing all content files at once.
You may use the css class docfix-hidden to hide your destination to the viewer. If someone searches for your destination or just clicks on a cross reference the document will still be scrolled to the location of your destination.
Do not use the css class hidden for hiding your destination tags, because this will crash the scrolling.

How crossreferencing works

DocFix will start searching for the requested target in the first content file you defined. On success, that means if the requested target was found inside the content file, the content file will be displayed and scrolled to the requested target.

If the requested target id was not found inside the first content file, DocFix searches the second content file and so on.

Please take care of targets with identical target ids. Every search action will start at the first defined content file. Therefore all targets with the same id that were defined after the first match will never be found / displayed.

What is a Navtag?

Can you see the list or navigation bar on the left?

The content of this navigation bar can be defined by setting up some (hidden) hyperlinks.

During the document parsing process DocFix searches for this tags and adds them to the navigation bar. If you can see, the position of the document gets updated when the user scrolls through your document.

The name of the first navtag can be defined inside the configuration section. Please look the documentation of the configuration options in chapter one for details.

A navigation bar with navtags will only be shown, if at least one navtag was defined.
